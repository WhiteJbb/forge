# Review Checklist — 완료 전 점검 목록

> 작업(PR) 완료 선언 전에 점검한다. 해당 없는 항목은 건너뛴다.

## 모든 PR 공통

- [ ] Plan.md 범위 안의 작업인가 (벗어나면 Approval Gate)
- [ ] 테스트 통과 (`test_forge.py` + 해당 작업의 단위 테스트)
- [ ] 실패 경로 테스트 포함 — 새 기능이 429/5xx/timeout/취소에서 어떻게 동작하는지
- [ ] 문서 정합성: DESIGN.md와 구현이 어긋나지 않는가 (어긋나면 문서 먼저 갱신)
- [ ] WorkLog.md 기록, Plan.md 상태 갱신, 의사결정 있었으면 DecisionLog.md
- [ ] PR 본문에 변경 요약 + 검증 결과 포함, squash merge

## 게이트웨이 특화 점검

- [ ] **OpenAI 호환성**: 표준 클라이언트(openai SDK)로 스트리밍/논스트리밍 양쪽 동작 확인
- [ ] **failover 투명성**: 클라이언트 입장에서 재시도가 보이지 않는가
- [ ] **요청 경로 블로킹 없음**: 메트릭/로깅이 동기 I/O를 하지 않는가
- [ ] **스트리밍**: usage 청크 처리, mid-stream 에러의 SSE 이벤트 형식
- [ ] **취소**: 클라이언트 disconnect 시 업스트림 취소되는가

## 보안/프라이버시

- [ ] API 키가 로그·에러 응답·대시보드에 노출되지 않는가 (마스킹)
- [ ] 프롬프트/응답 본문이 저장되지 않는가 (opt-in 로깅 제외)
- [ ] 새 엔드포인트에 인증 적용 여부 판단 (`/admin/*`은 loopback 한정)
- [ ] 기본값이 안전한가 (바인딩, 타임아웃, 인증)

## 릴리스 전 (마일스톤 완료 시)

- [ ] 실클라이언트 스모크 테스트: Cline 또는 Aider 연결해 실사용 1회 (M2부터 Claude Code 포함)
- [ ] `forge.yaml` 예시와 실제 스키마 일치
- [ ] README 연동 스니펫이 현재 동작과 일치
- [ ] CHANGELOG 갱신, 버전 번호 (semver)
