# Personas — 사용자 페르소나

> Forge가 상대하는 사용자 유형. 기능 우선순위 판단의 기준.

## P1. 김코더 — 무료 티어 개인 개발자 (핵심 타깃)

- **누구**: 사이드 프로젝트를 하는 개인 개발자. Cline/Aider를 매일 쓰지만 API 비용이 부담.
- **현재 행동**: NVIDIA 무료 티어 + OpenRouter 무료 모델 + 로컬 Ollama를 수동으로 갈아끼움. 429가 나면 에이전트 설정을 열어 모델명을 바꾼다.
- **원하는 것**: "무료로, 끊기지 않고, 알아서 좋은 모델." 설정은 한 번만.
- **Forge가 주는 가치**: 선제 스로틀 + failover로 429 무감지, `allow_paid: false` 정책, 5분 설치.
- **성공 지표**: 설치 후 에이전트 설정을 다시 여는 빈도 → 0.

## P2. 박리드 — 소규모 팀의 개발 리드

- **누구**: 5인 팀에서 코딩 에이전트 도입을 주도. 팀원별로 제각각인 모델 설정과 비용을 통제하고 싶다.
- **현재 행동**: 팀 공용 프록시를 검토 중. LiteLLM은 설정이 무겁다고 느낌.
- **원하는 것**: 팀 공용 엔드포인트 하나, 작업 유형별 정책, 비용 가시성.
- **Forge가 주는 가치**: Policy Engine(YAML 정책), 대시보드 비용/사용량, 외부 바인딩 + API 키 인증.
- **주의**: 멀티 유저 관리(가상 키, 유저별 예산)는 의도적으로 범위 밖 (DESIGN.md 리스크 논의) — 이 페르소나의 요구 중 일부만 충족한다는 것을 인지.

## P3. 최탐구 — 셀프호스팅 취미가 / 얼리어답터

- **누구**: 홈랩에 Ollama를 돌리고 새 모델이 나오면 바로 시험. HN/GitHub에서 도구를 찾음.
- **현재 행동**: 모델별 성능을 스프레드시트로 비교. 라우팅 로직이 불투명한 도구는 불신.
- **원하는 것**: 왜 이 모델이 선택됐는지 설명 가능할 것, 로컬 모델과 클라우드 모델의 자유로운 조합, 뜯어볼 수 있는 코드.
- **Forge가 주는 가치**: `/v1/route/explain`, 스코어 공식 공개, capability 오버라이드, 텔레메트리 없음.
- **역할**: 오픈소스 초기 기여자·전파자가 될 가능성이 가장 높은 층.
